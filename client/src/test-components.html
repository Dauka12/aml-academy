Test 2: DropdownList_r5 Component

Expected data format:

    "componentName": "DropdownList_r5",
    "values": {
        "headers": "[{\"name\":\"Агенттік\",\"icon\":null},{\"name\":\"Мемлекеттік органдар\",\"icon\":null},{\"name\":\"Жеке сектор\",\"icon\":null}]",
        "items": "[{\"title\":\"Кураторлық\",\"text\":\"Кураторлық – ҚМС әрбір түрі үшін жедел өзара іс-қимыл жасайтын және үйлестіруді жүзеге асыратын куратор (ҚР ҚМА қызметкері) бекітілген\"},{\"title\":\"Call-орталық\",\"text\":\"Call-орталық – ҚМС арналған өтініштер мен консультацияларды өңдеу орталығы (ҚР ҚМА колл-орталығы «14-58»)\"}]"
    }
}

Analysis

The issue was:

  • RandomParagraph: Was receiving React Fragment objects instead of plain strings, causing "[object Object]" display
  • DropdownList_r5: Was not rendering due to missing or incorrect data parsing

Fixes Applied

  • Changed cleanAndFormatText() to cleanTextOnly() for RandomParagraph to pass plain strings
  • Added debug logging for DropdownList_r5 to track data processing
  • The RandomParagraph component internally uses processTextWithFormattingHTML() which was fixed to return HTML strings
